## Refactor Target: src/components/resource-form.tsx and src/components/resource-table.tsx
****Identified Structural Flaw:**** These files contain large monolithic components that exceed typical structural limits (over 300 lines each) and house tight coupling between UI layout logic, data fetching/submission, and state management.
****Impact on Maintainability:**** This structural overhead creates cognitive friction when modifying standard form or table behaviors. The lack of modular decomposition blocks granular unit testing and makes it difficult to reason about the components' discrete responsibilities.
****The Clean Architecture Blueprint:**** Refactor `src/components/resource-form.tsx` into a modular directory `src/components/resource-form/` with separated files for UI fields (`fields.tsx`), hook logic (`hooks/`), and the main component (`index.tsx`). Do the same for `src/components/resource-table.tsx` -> `src/components/resource-table/`. Apply proper semantic naming.
****Verification & Refactor Logic:**** Ensure each separated module has clear, self-documenting responsibilities. Run `npm run test` and `npm run lint` to guarantee that structural migrations preserve all existing functionality.
